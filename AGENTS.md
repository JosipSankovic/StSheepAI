# AGENTS.md

## Workflow Instructions
This project uses a structured "Context Engineering" workflow:
1. Research Phase → Save to `thoughts/shared/research/`
2. Plan Phase → Save to `thoughts/shared/plans/`
3. Implement Phase → One section at a time
4. Validate Phase → Run tests, generate reports

## Directory Structure
- `thoughts/shared/research/` - Research documents
- `thoughts/shared/plans/` - Implementation plans
- `thoughts/shared/validation/` - Validation reports

## Working Agreements
- Always save artifacts before ending a session
- Run tests after each implementation section
- Clear context between major phases using /new
